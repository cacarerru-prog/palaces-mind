# Palaces of the Mind

Локальная персональная система памяти для разработчика, использующего Claude Code каждый день.

Идея простая: Claude умный, но забывает всё после каждой сессии. А ты не хочешь объяснять одно и то же по двадцать раз. Этот сервис тихо сидит в трее, перехватывает твои сессии Claude Code через хуки, прогоняет их через LLM (Google Gemini), раскладывает по темам и подтемам в локальной SQLite, а потом сам подкладывает нужные куски обратно в контекст — когда ты пишешь похожий запрос. Получается «второй мозг», который растёт вместе с твоей работой и не утекает никуда наружу.

---

## Что это умеет

- **Автоматический захват сессий Claude Code.** Хуки `UserPromptSubmit` и `Stop` перехватывают каждый твой запрос и каждый ответ ассистента.
- **Структурирование знаний через LLM.** Фоновый процесс берёт «затихшие» диалоги, отправляет в Gemini, получает обратно набор узлов: тема → подтема → краткая суть + детали + ключевые слова.
- **Инжект памяти обратно в Claude Code.** Когда ты пишешь новый запрос, хук ищет в базе релевантные узлы и аккуратно добавляет их в контекст под заголовком `🧠 Из твоей памяти`. Ты ничего не делаешь оно само.
- **Семантический поиск.** Помимо текстового совпадения работает поиск «по смыслу» через эмбеддинги Gemini (`text-embedding-004`). Можно искать «как Docker монтирует папки» и получать узел про `bind mount`, даже если слова в нём другие.
- **Гибридный режим поиска.** По умолчанию складывает текстовое совпадение и косинусную близость — лучшие результаты по обоим направлениям.
- **Десктопное приложение в трее.** Окно похоже на Obsidian: три колонки (фильтры/темы → список карточек → читалка). Копируется одной кнопкой что угодно: суть, детали, ключевые слова, весь узел в Markdown, отдельный блок кода. Сортировка по дате/теме/релевантности, фильтр по темам.
- **Single instance + AppUserModelID.** Можно закрепить на панели задач Windows, повторный запуск открывает уже работающее окно, крестик `✕` сворачивает в трей вместо выхода.
- **Маскировка секретов.** Перед записью в БД из транскриптов вычищаются токены и ключи (Google, OpenAI, Anthropic, GitHub, GitLab, Slack, AWS, NVIDIA, JWT, PEM, `password=`/`token=`).
- **Полноценный десктопный `.exe`.** FastAPI-бэкенд упакован в `palaces-server.exe` через PyInstaller и вшит прямо в NSIS-установщик. После установки Python в системе нужен **только для хуков Claude Code** — сам сервер автономный.

### Почему так

- **Хук `on_stop` не зовёт LLM напрямую.** Если бы звал — каждый ход диалога = 1 запрос к Gemini, бесплатный лимит съедался бы за час. Вместо этого хук дёшево сохраняет сессию, а фоновый «уборщик» внутри FastAPI-сервера берёт сессии, которые «затихли» дольше 5 минут, и разбирает каждую один раз. Один диалог = один запрос к LLM.
- **Хук `on_message` ходит только в SQLite.** LLM при инжекте контекста не вызывается вообще — это быстро и не сжигает лимит.
- **Семантический поиск считается в Python.** Без векторных движков типа sqlite-vec: для нескольких тысяч узлов косинус на чистом Python укладывается в десятки миллисекунд.
- **Все запросы к API защищены токеном.** Electron при старте генерирует случайный `API_TOKEN`, передаёт его и Python-серверу (через `PALACES_TOKEN`), и React-окну (через preload-аргумент). Middleware FastAPI требует заголовок `X-Palaces-Token` — посторонний сайт его не знает.

## Установка

Есть два пути: **поставить готовый инсталлятор** (просто пользоваться) или **собрать самому из исходников** (хочешь править код).

### Путь 1 — готовый `.exe` (рекомендуется)

1. Возьми `Palaces of the Mind Setup 1.0.0.exe` из релизов (`frontend/release/` после сборки или из GitHub Releases, когда выложу).
2. Двойной клик → NSIS oneClick установит приложение в `%LOCALAPPDATA%\Programs\Palaces of the Mind\` и сразу запустит его.
3. В меню Пуск появится ярлык — правый клик → **«Закрепить на панели задач»**.
4. В трее появится значок. Меню по правому клику: Открыть / Запускать при входе в Windows / Выйти.

Внутри инсталлятора:
- Electron + React-сборка (UI);
- `palaces-server.exe` — автономный FastAPI-сервер, упакованный PyInstaller'ом (включает uvicorn, openai, sqlite3, fastapi и все остальные зависимости).

После установки **Python в системе нужен только для хуков Claude Code** (`on_message.py`, `on_stop.py`). Сам сервер — самостоятельный `.exe`.

Дополнительно нужны:
- **API-ключ Gemini** в `D:\mind\palaces-of-the-mind\backend\.env`:
  ```
  LLM_API_KEY=твой_ключ_сюда
  ```
  Бесплатный ключ — в [Google AI Studio](https://aistudio.google.com/apikey). Файл `.env` в `.gitignore` и в репозиторий не попадёт.

- **Хуки Claude Code** в `C:\Users\<твоё_имя>\.claude\settings.json`:
  ```json
  {
    "hooks": {
      "UserPromptSubmit": [{
        "matcher": "",
        "hooks": [{
          "type": "command",
          "command": "python D:\\mind\\palaces-of-the-mind\\backend\\palaces\\hooks\\on_message.py"
        }]
      }],
      "Stop": [{
        "matcher": "",
        "hooks": [{
          "type": "command",
          "command": "python D:\\mind\\palaces-of-the-mind\\backend\\palaces\\hooks\\on_stop.py"
        }]
      }]
    }
  }
  ```

База `memory.db`, логи и `.env` лежат в `D:\mind\palaces-of-the-mind\` независимо от установленного `.exe`. Поэтому переустановка приложения данные не теряет.

### Путь 2 — сборка из исходников

1. **Python-зависимости:**
   ```powershell
   cd D:\mind\palaces-of-the-mind\backend
   pip install -r requirements.txt
   pip install pyinstaller
   ```

2. **Node-зависимости:**
   ```powershell
   cd D:\mind\palaces-of-the-mind\frontend
   npm install
   ```

3. Создай `.env` с ключом Gemini и пропиши хуки (см. Путь 1).

## Запуск и сборка

### Режим разработки

```powershell
cd D:\mind\palaces-of-the-mind\frontend
npm run dev
```

Vite поднимает dev-сервер на `http://localhost:5173`, Electron спавнит Python-бэкенд (`python -m api.main` → FastAPI на `127.0.0.1:8765`) и открывает окно. В трее появляется иконка.

Чтобы в этом режиме сработал `npm run build` чуть позже — нужен ещё одноразовый шаг для PyInstaller-кэша (см. ниже).

### Сборка инсталлятора `.exe`

Делается в два этапа: сначала собираем автономный бэкенд через PyInstaller, потом упаковываем всё через electron-builder.

```powershell
# 1. Собрать palaces-server.exe (~28 МБ, в backend/dist/)
cd D:\mind\palaces-of-the-mind\backend
python -m PyInstaller palaces-server.spec --clean

# 2. Скопировать его в frontend/resources/ — оттуда electron-builder его подхватит
copy backend\dist\palaces-server.exe frontend\resources\palaces-server.exe

# 3. Собрать установщик (~101 МБ, в frontend/release/)
cd D:\mind\palaces-of-the-mind\frontend
npm run build
```

Готовый установщик: `frontend/release/Palaces of the Mind Setup 1.0.0.exe`.

#### Подводные камни сборки

- **macOS-симлинки в winCodeSign.** Electron-builder качает архив `winCodeSign-2.6.0.7z` и при распаковке падает на macOS-симлинках `.dylib`, потому что Windows без прав админа или Developer Mode симлинки создавать не умеет. **Обход:** распаковать кэш вручную, исключая папку `darwin`:
  ```bash
  CACHE=C:/Users/<username>/AppData/Local/electron-builder/Cache/winCodeSign
  mkdir -p "$CACHE"
  curl -L -o "$CACHE/winCodeSign-2.6.0.7z" https://github.com/electron-userland/electron-builder-binaries/releases/download/winCodeSign-2.6.0/winCodeSign-2.6.0.7z
  "frontend/node_modules/7zip-bin/win/x64/7za.exe" x "$CACHE/winCodeSign-2.6.0.7z" -o"$CACHE/winCodeSign-2.6.0" -y -x'!darwin'
  ```
  После этого `npm run build` подхватит кэш и не будет качать архив повторно.

- **Антивирус ругается на `palaces-server.exe`.** PyInstaller-сборки часто триггерят ложные срабатывания Defender'а — он видит запакованный Python и пугается. Добавь исключение для папки `frontend/resources/` и установленного приложения.

- **Свободное место на C:.** NSIS-установщик распаковывает архив в `%TEMP%` и устанавливает приложение в `%LOCALAPPDATA%`. Нужно минимум ~250 МБ свободного места на системном диске.

### Полезные команды

| Команда | Что делает |
|---------|-----------|
| `npm run dev` | Vite + Electron + Python-бэкенд (для разработки) |
| `python -m PyInstaller palaces-server.spec --clean` | Пересобрать автономный сервер `palaces-server.exe` |
| `npm run build` | Собрать NSIS-установщик `Palaces of the Mind Setup 1.0.0.exe` |
| `cd backend && python -m api.main` | Запустить сервер вручную (без Electron) на порту 8765 |

## Возможности интерфейса

### Главная страница «Библиотека»

Три колонки:

- **Слева** — поиск, сортировка, список тем с количеством узлов. Клик по теме — фильтр. Внизу кнопка `⚡ Проиндексировать память` достраивает эмбеддинги для старых записей.
- **В центре** — карточки узлов, которые подходят под фильтры. Hover показывает кнопку копирования сути.
- **Справа** — читалка выбранного узла: суть, подробности с подсветкой блоков кода, ключевые слова. Каждый блок копируется отдельной кнопкой, всю запись можно скопировать целиком в формате Markdown. Есть режим правки (PATCH) и удаление.

### Поиск

Три режима, переключаются под полем ввода (появляются, когда есть запрос):

- **Текст + смысл** (по умолчанию) — гибрид: текстовое совпадение и косинусная близость складываются с весами 0.55 / 0.45.
- **По словам** — только LIKE по `summary` / `detail` / `keywords`, сортировка по числу совпадений (summary весит ×3, keywords ×2, detail ×1).
- **По смыслу** — только эмбеддинги. Полезно когда не помнишь точных слов.

Совпадения подсвечиваются жёлтым в превью карточек и в читалке.

### Сортировка

Релевантность (только в поиске), сначала новые, сначала старые, недавно изменённые, по теме.

### Копирование

Кнопка `⧉` рядом с каждым полем (суть, детали, ключевые слова), отдельная кнопка на каждом блоке кода (по `hover`), и большая кнопка `⧉ Копировать всё` сверху читалки — копирует весь узел в виде Markdown:

```markdown
# Тема → Подтема

Краткая суть в одном-двух предложениях.

Подробное объяснение со всеми деталями и блоками кода.

Keywords: keyword1, keyword2, keyword3
```

## Безопасность

- **API-ключ Gemini** хранится в `.env` (в `.gitignore`).
- **FastAPI открыт только на 127.0.0.1.** Из внешней сети сервер не виден.
- **Каждый запрос к API** требует заголовок `X-Palaces-Token`. Токен генерируется заново при каждом запуске Electron и передаётся серверу через переменную окружения, а окну — через `additionalArguments`. Подделать его браузер не сможет, потому что он не знает значения. Если сервер запущен вручную без токена — проверка отключается (для тестов).
- **CORS** разрешает только `http://localhost:5173` (Vite) и `null` (file:// в собранном Electron). Звёздочку `*` не используем — иначе любой открытый в браузере сайт мог бы из JS читать твою базу знаний.
- **Маскировка секретов.** Перед записью в БД из транскриптов и из запросов вычищаются: Google AI ключи (`AIza…`), OpenAI (`sk-…`), Anthropic (`sk-ant-…`), GitHub (`ghp_…`, `github_pat_…`), GitLab (`glpat-…`), Slack (`xox*-…`), AWS Access Key (`AKIA…`), NVIDIA NIM (`nvapi-…`), JWT (`eyJ…`), PEM-блоки приватных ключей, пары `password=`/`token=`/`secret=`, длинные «подозрительные» токены ≥ 40 символов со смесью букв и цифр.
- **`memory.db`, `logs/`, `diploma/`, `.env`** — всё в `.gitignore`. Только код уходит в репозиторий.

## Особенности реализации

### Кириллица в SQLite

Встроенная функция `LOWER()` в SQLite умеет приводить к нижнему регистру только латиницу. Поэтому поиск `"Горутина"` не находил `"горутина"`. Решение: переопределили `lower()` через `conn.create_function("lower", 1, str.lower)` — теперь питоновский `str.lower()` корректно работает с любой Unicode-строкой.

### UTF-8 в хуках на Windows

По умолчанию `sys.stdin`/`stdout`/`stderr` на Windows работают в `cp1251`, которая не умеет эмодзи. `print('🧠')` падал с `UnicodeEncodeError`. Решение — в самом начале каждого хука принудительный `reconfigure(encoding="utf-8")`.

### Хуки Claude Code

В ранней документации упоминалось событие `UserMessage` — его на самом деле не существует. Реальное событие — `UserPromptSubmit`, и поле prompt лежит сразу в корне JSON. Событие `Stop` присылает не транскрипт целиком, а `transcript_path` — путь к `.jsonl` файлу, который нужно прочитать самим. Инжект контекста — через `hookSpecificOutput.additionalContext`, не как корневое поле.

### Single instance + AppUserModelID

Windows группирует кнопки на панели задач по `AppUserModelID`. Без явной установки Electron-приложения часто отображаются как `electron.exe` и плохо пинятся. Указываем `app.setAppUserModelId('com.palaces.mind')` — тот же, что в `electron-builder.yml`. Плюс `app.requestSingleInstanceLock()` — второй запуск завершается, у первого срабатывает событие `second-instance`, и мы показываем уже открытое окно.
